# ck benchmarks

Quality and speed measurements for ck's semantic search, run against the same dataset semble published.

## Attribution

This benchmark **reuses semble's dataset** ([MinishLab/semble](https://github.com/MinishLab/semble), MIT):

- `repos.json` — 63 repos across 19 languages with pinned revisions
- `annotations/*.json` — ground-truth query → relevant-file labels (queries + labels generated by Claude Sonnet 4.6, verified by LLM-as-judge per semble's methodology)

Using their dataset gives us apples-to-apples comparison with their published numbers.

## Dataset shape

| Category | Queries | What it tests |
|---|---:|---|
| semantic | 711 | Code that implements a specific behavior or concept |
| architecture | 343 | Design decisions, module boundaries, structural patterns |
| symbol | 204 | Named entity lookup (function, class, type, variable) |

19 languages: bash, C, C++, C#, Elixir, Go, Haskell, Java, JavaScript, Kotlin, Lua, PHP, Python, Ruby, Rust, Scala, Swift, TypeScript, Zig.

## Metric

NDCG@10 at file level — exactly what semble used for ripgrep/probe/grepai/colgrep baselines. We don't measure span-level matching (which semble does only for itself); file-rank is the apples-to-apples number to compare with their published per-language tables.

## Running

```bash
# Clone the benchmark repos (pinned revisions, into ~/.cache/ck-bench)
python3 benchmarks/sync_repos.py

# Run on one or a few repos (good for iteration)
python3 benchmarks/baselines/ck.py --repo curl --repo fastapi

# Run by language
python3 benchmarks/baselines/ck.py --language python

# Full run (slow — clones + indexes all 63 repos)
python3 benchmarks/baselines/ck.py
```

Results land in `benchmarks/results/ck-<sha12>.json`.

Requires the `ck` binary on `$PATH` (`cargo install ck-search --locked` or `npm install -g @beaconbay/ck-search`). Tests against your installed version, prints it in the header.

## Out of scope (for now)

semble's published results also include token-efficiency vs grep+read at fixed recall budgets. We don't run that here yet — once we've validated NDCG numbers, that's a natural follow-up.

The dataset is **code-only**. ck's longer-term vision is mixed-content search — PDFs, docs, configs — which this benchmark doesn't measure. A separate mixed-content benchmark is on the roadmap.
